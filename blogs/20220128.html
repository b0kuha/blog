    # 常用命令

    • 拉取分支

    将远程仓库的分支拉取到本地，可以改名，但不建议

    git fetch origin 远程分支名:本地分支名

    • 拉取代码

    推送代码前记得先拉取代码到最新

    git pull origin 远程分支名

    git push origin 远程分支名

    # 基本使用步骤

    1. 创建一个工作目录并初始化 git
    # 1. 创建文件夹并进入
mkdir workspace && cd workspace
# 2. 使用git初始化文件夹
git init
    1
    2
    3
    4
    1. 新建文件，将其加入 git 缓存区
    # 创建一个文件
touch index.html
# 加入缓存区
git add index.html
    1
    2
    3
    4

    加入缓存区常用指令(将所有修改过的文件加入缓存区)

    git add -A
# OR
git add .
    1
    2
    3
    1. 提交到本地仓库
    git commit -m '初始化提交'
    1
    1. 推送到远程开发分支
    # 推送代码前记得先拉取远程代码
git pull origin development
# 推送到远程分支
git push origin development
    1
    2
    3
    4

    在实际开发中经常需要将自己的分支合并到开发分支再推送

    例如：团队中的远程开发分支为 development，本地以 development 创建分支

    # 以development为基础创建本地开发分支
git checkout -b dev.pc.xxx
# 在dev.pc.xxx分支下开发需求并commit到本地仓库
git add .
git commit -m '新功能开发完毕'
# 切换到development分支
git checkout development
# 拉取最新代码并合并我们自己的分支
git pull origin development
git merge --no-ff dev.pc.xxx
git push	origin development
